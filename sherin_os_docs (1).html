üîë ID System Architecture

IDs are lightweight pointers (~50 bytes) that travel through the network while actual data stays in SHFS storage.

Area ID

Pattern: A_<LLLL>:<RRR>:<SSS>:<VVV>:<VV>
Example: A_0001:001:004:023:12

Components:
‚Ä¢ A_ = Area prefix
‚Ä¢ LLLL = Layer number (zero-padded)
‚Ä¢ RRR = Ring number
‚Ä¢ SSS = Slice number (1-12)
‚Ä¢ VVV = Sub-area number
‚Ä¢ VV = Vertex within sub-area

Bot ID

Pattern: B_<LLLL>:<RRR>:<SSS>:<VVV>:<VV>
Example: B_0001:001:004:023:12

Bot ID mirrors its assigned Area ID ‚Äì only the prefix changes.

Task ID

Pattern: tk_<PPP>:<SS>
Example: tk_002:01

Components:
‚Ä¢ PPP = Priority (001=critical, 002=high, 003=mid, 004=low)
‚Ä¢ SS = Sequence within priority

Request ID

Pattern: req_<LLL>:<SS>
Example: req_005:01

Links multiple stages of the same logical request.

Token ID

Pattern: TK_<AREA_ID>:<CHK>
Example: TK_A_0001:001:004:023:12:a89ef60e

Components:
‚Ä¢ AREA_ID = Full Area ID where token lives
‚Ä¢ CHK = 8-character SHA-256 checksum

ID to Data Conversion

When a bot needs actual data, it performs a fast lookup:

  1. Receive ID ‚Üí read storage/index.json
  2. Get absolute file path from index
  3. Open file ‚Üí parse JSON ‚Üí obtain full data object
  4. Return object to requesting bot

Total time: <10ms with 85% cache hit rate

üîÑ Data Flow Architecture

User ‚Üí Entry Layer ‚Üí Layer 1 ‚Üí Layer 2 ‚Üí Layer 3 ‚Üí UI

ID Exits Current Layer

Task completes processing at current layer and ID exits via nearest port to outer orbital ring.

Outer Ring Transit

ID travels counter-clockwise on outer ring towards the central hub. No payload data travels, only the ~50 byte ID.

Hub Routing Decision

Core hub (0,0,0) reads target Area ID from metadata and selects destination layer/port combination.

Inner Ring Entry

ID enters inner ring moving clockwise towards target port. Opposite direction prevents collisions.

Layer Entry

ID arrives at destination layer via chosen port. Bot assigned to that area receives the ID.

Data Retrieval

Bot looks up ID in SHFS index, fetches actual data from storage, and begins processing.

Bandwidth Optimization

By moving only IDs instead of full payloads:

  • Network traffic reduced by 40:1 compression ratio
  • Each transfer is ~100 bytes vs. ~2KB+ for full data
  • Multiple IDs can be batched in single transit
  • Ring congestion is minimized
